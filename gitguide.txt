깃의 버전관리 시스템
1. 작업트리
2. 스테이지
3. 저장소(repository)

내가 만든 디렉토리가 작업트리가 된다.
스테이지는 버전으로 만들 파일이 대기하는 곳이다.
저장소는 스테이지에서 대기하던 파일을 버전으로 만들어 저장하는 곳이다.

정리하자면,
작업트리에서 문서를 수정하고 수정한 파일중 버전으로 만들고 싶은 파일을 스테이지에 올린다. 그리고 스테이지에 있던 파일을 저장소로 커밋하는 것이 깃이 버전을 만드는 순서이다.

git add 파일명.확장자를 통해 스테이징 하고
git commit -m "커밋하면서 올릴 메세지 내용"을 통해 저장소에 커밋한다.

git status 로 깃의 상태를 확인할 수 있고
git log로 저장소에 저장된 버전을 확인할 수 있다.
git diff로 작업트리에 있는 파일과 스테이지에 있는 파일을 비교하거나, 스테이지에 있는 파일과 저장소에 있는 최신커밋을 비교해서 수정한 파일을커밋하기 전에 최종적으로 검토할 수 있다.
